Replica Sets : Constantly monitors your pods and forces the exact number of copies you want to stay online. If one crashes, it instantly starts an identical replacement.

Deployments :Manages the update process. It slowly replaces old pods with new versions one-by-one so your website never goes down during an update.

Stateful Sets :Manages pods that must remember data (like databases). It starts them in a strict order and locks each one to a permanent name and dedicated storage drive.

Daemon Sets :Automatically deploys exactly one copy of a pod to every single server in your network. It is used for background tasks like security monitoring.
